js事件循环机制

js是单线程，为了利用多核CPU,h5提出web worker标准。允许创建多线程，但不得操作Dom

Javascript执行引擎之外，有一个任务队列，当在代码中调用webAPI时，会交给浏览器内核的其他模块进行处理(以webkit为例，是webcore模块），当到到达触发条件（以setTimeout为例）即到达设置的延时时间时，这一延时方法被添加至任务队列里等到调用栈中的task执行完之后再回去执行任务队列之中的task
（1）所有同步任务都在主线程上执行，形成一个执行栈（execution context stack）。
（2）主线程之外，还存在一个”任务队列”（task queue）。只要异步任务有了运行结果（如AJAX,setTimeout），就在”任务队列”之中放置一个事件。
（3）一旦”执行栈”中的所有同步任务执行完毕，系统就会读取”任务队列”，看看里面有哪些事件。那些对应的异步任务，于是结束等待状态，进入执行栈，开始执行。
（4）主线程不断重复上面的第三步。（循环不断，因此这个机制成为事件循环 Event Loop）

深入：
任务队列分为macro-task和mic-rotask）
micro-tasks:process.nextTick，promise的then方法，Object.observe，MutationObserver
macrotasks:setTimeout，setInterval，setImmediate，I/O，UI渲染（setTimeout作为一个任务分发器是立即执行的，而回调函数延迟执行进入任务队列）


  • JavaScript代码的执行过程中，除了依靠函数调用栈来搞定函数的执行顺序外，还依靠任务队列(task queue)来搞定另外一些代码的执行。

  • setTimeout/Promise等我们称之为任务源。而进入任务队列的是他们指定的具体执行任务。

  • 一个线程中，事件循环是唯一的，但是任务队列可以拥有多个。（不同的任务源进去不同的执行队列，nextTick队列比promise队列先执行）

  • 事件循环的顺序，决定了JavaScript代码的执行顺序。它从script(整体代码)开始第一次循环。之后全局上下文进入函数调用栈。直到调用栈清空(只剩全局)，然后执行所有的micro-task。当所有可执行的micro-task执行完毕之后。循环再次从macro-task开始，找到其中一个任务队列执行完毕，然后再执行下一个macro-task队列，然后执行所有的micro-task，这样一直循环下去。

js知识点

一。js内存泄漏

js采用垃圾回收机制，使用标记-清除算法–当变量进入环境，就对变量进行标记。然后去掉环境中的变量以及被应用的变量的标记，在此之后加上标记的变量将被视为要删除的变量


典型的js内存泄漏



  1. 全局变量：全局变量不会被自动回收，只能手动将变量指向null进行回收
    1.未使用var而申明的变量
    2.错误使用this 导致的全局变量function foo() {
    this.variable = “potential accidental global”;}
  2. 被遗忘的计时器

  3. dom被清空时，未清除事件导致内存泄漏（Jquery会帮助清除事件）

  4. 闭包引起的内存泄漏

  5. 子元素存在引用，引用了子元素后，将父元素回收之后，子元素的引用任然存在。






二。事件代理


指将事件添加到本来要添加事件的父节点，将事件委托给父节点来触发处理函数。这得益于浏览器的事件冒泡机制


通过if(e.target && e.target.nodeName.toUpperCase == “LI”) { } 来判断执行事件的具体元素




三。类型判断


typeof操作符


typeof的返回值有undefined、boolean、string、number、object、function、symbol。对undefined 使用返回undefined、null使用返回object，NaN使用返回numbe


instanceof 操作符：判断已知对象的类型


obj instanceof Array
uindefined和null是object类型


constructor属性


Object.prototype.toString.call



var toString = Object.prototype.toString;
console.log(toString.call(new Date) === ‘[object Date]’);//true
console.log(toString.call(new String) ===’[object String]’);//true
console.log(toString.call(new Function) ===’[object Function]’);//true
console.log(toString.call(Type) ===’[object Function]’);//true

console.log(toString.call(true) ===’[object Boolean]’);//true

四。js严格模式


  1. 变量都必须先用var命令声明，然后再使用。

  2. 正常模式下，Javascript语言有两种变量作用域（scope）：全局作用域和函数作用域。严格模式创设了第三种作用域：eval作用域。

  3. 禁止this关键字指向全局对象

  4. 严格模式下无法删除变量。只有configurable设置为true的对象属性，才能被删除。

  5. 不可对对象的只读属性赋值，不可对对象的不可配置属性使用delete操作符，不可为不可拓展的对象添加属性，均抛TypeError

  6. 对象属性名必须唯一，函数中不可有重名参数

  7. 淘汰arguments.callee和arguments.caller

  8. 不可在if内部声明函数

  9. 禁止使用with语句

url知识总结

url：统一资源定位符，是uri的子集，uri包括url(通过位置定位资源)和urn(通过名字定位资源，标准还未成熟)

常见url组成：方案：//服务器位置/路径 例http://www.baidu.com/index


方案:告知客户端如何访问资源，为协议名，http，ftp等


服务器位置：服务器域名或ip地址


路径：服务器上资源的本地名，由/与前面的url路径分割


url语法


不同协议下的url略有不同，总体包括以下9个部分


scheme://user:password@host:port/path;params?query#frag


其中最主要的就是上文中的scheme方案，host服务器主机地址，path服务器资源位置
通过不同的分隔符来分离各个部分。
url有相对和绝对两种形式，其中相对url需要一个基础url进行解析。


url采用ASCII进行编码
通过转义来控制避免安全字符集的表示如%,~,?,#,/等。转义方法，%后面跟着两个ASCII码的16进制数。

webpack 搭建react环境

发表于 2017-11-03 |

npm 安装webpack react react-dom
然后配置webpack.config.js文件
这里要安装webpack-dev-server 监听文件改动并自动刷新浏览器
基本配置如下
module.exports = {
devtool: ‘eval-source-map’,//生成Source Maps,这里选择eval-source-map
entry: [‘webpack/hot/dev-server’, dirname + ‘/app/main.js’],//唯一入口文件,dirname是node.js中的一个全局变量，它指向当前执行脚本所在的目录
output: {//输出目录
path: __dirname + ‘/build’,//打包后的js文件存放的地方
filename: ‘bundle.js’//打包后输出的js的文件名
},

devServer: {
    contentBase: './build',//默认webpack-dev-server会为根文件夹提供本地服务器，如果想为另外一个目录下的文件提供本地服务器，应该在这里设置其所在目录（本例设置到"build"目录）
    colors: true,//在cmd终端中输出彩色日志
    historyApiFallback: true,//在开发单页应用时非常有用，它依赖于HTML5 history API，如果设置为true，所有的跳转将指向index.html
    inline: true,//设置为true，当源文件改变时会自动刷新页面
    port: 8080,//设置默认监听端口，如果省略，默认为"8080"
    process: true,//显示合并代码进度
}

在package.json中配置启动webpack-dev-server命令
{
…
“scripts”: {
“start”: “webpack”,
“dev”:”webpack-dev-server”
},
…
}
输入npm run dev 启动webpack-dev-server服务器

接下来设置loder，添加webpck.config.js文件如下
module:{
rules:[{
test:/(.js|.jsx)$/,//匹配loders处理文件的扩展名的正则表达式
use:{
loader:”babel-loader”,
options:{
presets:[//注意是数组，添加不同的loader
“es2015”,”react”
],
} //options 可以放在.babelrc文件中
},
exclude:/node_modules/, //屏蔽不需要处理的文件
}]
}

这样已经配置了一个简单的react基础工程了，可以开始愉快的学习react了

附：可以用官方的create-react-app 搭建项目，链接https://github.com/facebookincubator/create-react-app#getting-started

鼠标悬浮效果

发表于 2017-10-26 |

实现效果：https://ww3.sinaimg.cn/large/006tNbRwly1fcr5jmrmujg30cj06xqv5.gif

技术点
1.css3 filter属性
filter: blur(5px) 给元素设置高斯模糊
整个元素都会有模糊效果，对背景色image 和 图片img都有效果
实现背景模糊，内容清晰 设置两个元素，然后用绝对定位重叠来实现

2.内容区的 边框 从中间展开的效果
分别设置四条边为单独的元素
以上边框为例 开始长度为0，left为50% 利用hover时，增加长度为100%，left为0，
加上transtion实现渐变效果
transition 写在主要 样式中 ，不写在hover中 ，可以实现悬浮和离开时都有渐变效果，写在hover中 就只有 悬浮时有效果
transition: all 0.7s ease transition应用的元素一般未widh等长度元素 和color等颜色元素。

3.文字的流光渐变动画效果
文字渐变效果，原理是 背景色设置为渐变色，然后添加文字，通过使文字变为透明色，露出背景色来实现渐变效果的字体颜色（无法直接给文字color设置为渐变色）

(1)颜色渐变效果
-webkit-linear-gradient(left, #147B96, #E6D205, #147B96, #E6D205, #147B96);
第一个参数top为从上倒下，left为从左到右 后面的颜色参数表示渐变颜色。

(2)background-clip属性 值content-box、padding-box 或 border-box 裁剪背景图像到指定盒子大小

-webkit-background-clip: text; 保留文字部分的背景，裁剪掉其余部分的背景。

(3)-webkit-text-fill-color: transparent; 使文字为透明色，露出背景色，实现镂空效果
直接适用color：transparent 文字就消失

(4)实现动画效果
设置css动画使 背景滚动，实现颜色渐变的动画
@keyframes masked-animation {
0% { background-position: 0 0;}
100% { background-position: -100% 0;}
}

然后对于元素使用动画
animation：masked-animation 7s infinite;

注意要设置 background-size:200% 100%; 背景扩大一倍 ，背景才有移动的空间，不然背景无法移动，不能实现动画效果。
其他属性 数值（直接设置宽度和高度）
cover 此属性有高度或宽度100%拉伸的效果，也就是容器多大，背景图片最短的那一边就会拉伸多大。
contain 最长的边 拉升到容器大小
